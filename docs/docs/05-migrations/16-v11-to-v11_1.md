# Migrating from v11.0 to v11.1

This guide provides instructions for migrating to a new version of ibc-go.

**Note:** ibc-go supports golang semantic versioning and therefore all imports must be updated on major version releases.

## Packet Forwarding Middleware

The packet forwarding middleware has been moved from [cosmos/ibc-apps](https://github.com/cosmos/ibc-apps) to ibc-go in this release. The packet forwarding middleware is now available under `modules/apps/packet-forward-middleware`. 

Your migration path will depend on how you have integrated the packet forwarding middleware in your application prior to this release. If you've been using the packet forwarding middleware from ibc-apps, you're likely upgrading from v10.0 to v11.1 and should follow the instructions in the [migration guide for v10 to v11.1](./17-v10-to-v11_1.md). 
In this guide, we will assume that you have not been using the packet forwarding middleware from ibc-apps and are upgrading from v11.0 to v11.1.

If you wish to use the packet forwarding middleware, you will first need to follow the [integration instructions](../04-middleware/02-packet-forward-middleware/02-integration.md). 

### Add `StoreUpgrades` for PFM module

For PFM it is also necessary to [manually add store upgrades](https://docs.cosmos.network/sdk/latest/guides/upgrades/upgrade#adding-new-modules-during-an-upgrade) for the new PFM module and then configure the store loader to apply those upgrades in `app.go`:

```go
if upgradeInfo.Name == "v11_1" && !app.UpgradeKeeper.IsSkipHeight(upgradeInfo.Height) {
  storeUpgrades := store.StoreUpgrades{
    Added: []string{packetforwardtypes.StoreKey},
  }

  app.SetStoreLoader(upgradetypes.UpgradeStoreLoader(upgradeInfo.Height, &storeUpgrades))
}
```

This ensures that the new module's stores are added to the multistore before the migrations begin.
If a chain chooses not to integrate the PFM module, it does not need to add the PFM key to the `Added` field.
